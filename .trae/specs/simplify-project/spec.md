# NyxAI 项目简化 - 产品需求文档

## 概述
- **摘要**：对 NyxAI 项目进行简化，使 PostgreSQL 变为可选，支持 SQLite，向量数据库使用 Chroma，移除 Redis 依赖。
- **目的**：降低项目部署和运行的复杂性，减少外部依赖，使项目更加轻量和易于部署。
- **目标用户**：开发人员和系统管理员，希望快速部署和运行 NyxAI 系统。

## 目标
- 将 PostgreSQL 数据库设置为可选，默认使用 SQLite
- 确保系统在使用 SQLite 时能正常运行
- 确认向量数据库使用 Chroma 作为默认选项
- 移除 Redis 依赖，使用内存缓存或文件系统缓存替代
- 确保所有功能在简化后仍然正常工作

## 非目标（范围外）
- 不修改核心业务逻辑
- 不改变系统的 API 接口
- 不影响现有的监控和告警功能
- 不修改前端代码

## 背景与上下文
- 当前项目依赖 PostgreSQL、Redis 和向量数据库（如 pgvector）
- 这些依赖增加了部署和维护的复杂性
- 对于小型部署或开发环境，这些外部依赖可能是不必要的
- 项目已经支持 SQLite 作为嵌入式数据库选项
- 项目已经集成了 Chroma 作为向量数据库选项

## 功能需求
- **FR-1**：PostgreSQL 变为可选，默认使用 SQLite
- **FR-2**：移除 Redis 依赖，使用内存缓存或文件系统缓存替代
- **FR-3**：确认向量数据库使用 Chroma 作为默认选项
- **FR-4**：确保系统在简化后能正常启动和运行

## 非功能需求
- **NFR-1**：系统启动时间不超过 10 秒
- **NFR-2**：内存使用不超过 512MB（在默认配置下）
- **NFR-3**：系统应在没有外部依赖的情况下运行

## 约束
- **技术**：Python 3.11+，FastAPI，SQLAlchemy，ChromaDB，uv 依赖管理工具
- **业务**：保持系统的核心功能不变
- **依赖**：移除 Redis 依赖，使 PostgreSQL 变为可选

## 假设
- 系统在使用 SQLite 时性能满足需求
- 内存缓存或文件系统缓存足以替代 Redis
- Chroma 作为向量数据库能满足系统需求

## 验收标准

### AC-1：PostgreSQL 变为可选
- **给定**：系统配置为使用 SQLite
- **当**：启动系统时
- **然后**：系统正常启动，使用 SQLite 数据库
- **验证**：`programmatic`

### AC-2：Redis 依赖被移除
- **给定**：系统在没有 Redis 的环境中运行
- **当**：启动系统时
- **然后**：系统正常启动，不依赖 Redis
- **验证**：`programmatic`

### AC-3：向量数据库使用 Chroma
- **给定**：系统配置为使用 Chroma 作为向量数据库
- **当**：系统运行时
- **然后**：向量搜索功能正常工作
- **验证**：`programmatic`

### AC-4：系统功能正常
- **给定**：系统在简化配置下运行
- **当**：执行各种系统功能时
- **然后**：所有功能正常工作
- **验证**：`programmatic`
